# 구현 계획서 (분석 및 통계 탭 고도화 및 수금 상태 동적 관리)

본 구현 계획서는 분석 및 통계 탭의 완성도를 끌어올리기 위해 **1) 주간/일간 차트 연동 버그 수정, 2) 도넛 차트 비중 비율(%) 렌더링, 3) 수금/미수금 상태의 로컬 스토리지 관리 및 대시보드 내 수동 정산 관리 제어 기능 추가**를 반영하고자 작성되었습니다.

---

## User Review Required

> [!IMPORTANT]
> - **도넛 차트 비중 비율(%) 노출**: 품목별 판매 비중을 나타내는 도넛 차트의 라벨 및 툴팁에 실제 퍼센티지(예: `골드 감자 (55%)`)가 직관적으로 노출되도록 개선합니다.
> - **주간/일간 차트 동적 연동**: 버튼을 클릭해도 차트가 바뀌지 않던 문제를 수정하기 위해 `updateChartType` 함수를 신설하고, 월간/주간/일간 데이터를 Chart.js에 동적으로 주입하도록 연동합니다.
> - **수금 및 미수금 정보 데이터화 및 수동 관리**:
>   - 현재 수금/미수금 데이터는 `app.js`의 `ANALYTICS_DATA.buyers` 객체 내에 하드코딩되어 있습니다.
>   - 이를 해결하기 위해, 거래처별 정산 수금 데이터를 `localStorage`(`bb_analytics_buyers`)에 저장 및 관리할 수 있는 저장소 클래스를 확장합니다.
>   - 분석 및 통계 탭 하단의 '거래처별 구입 누적 현황' 테이블에 사장님이 직접 수금 상태를 `[정상 (수금완료)]`, `[미수금]`으로 토글할 수 있는 수동 조작 드롭다운을 제공하여, 실무적인 입금 확인 프로세스를 시뮬레이션할 수 있게 합니다.

## Proposed Changes

### [app.js]
- **거래처 통계 및 수금 데이터 로컬스토리지 연동 (`BongBongStore` 확장)**:
  - `localStorage`에 `bb_analytics_buyers` 키로 수금 데이터를 관리하는 메소드 추가.
  - 초기값으로 기존 `ANALYTICS_DATA.buyers` 데이터를 적재.
  - 특정 거래처의 수금 상태를 변경하는 `updateBuyerStatus(buyerName, newStatus)` 메소드 구현.

### [index.html]
- **분석 탭 차트 컨트롤러(`updateChartType`) 구현**:
  - 월간/주간/일간 탭 버튼 클릭 시 활성화 클래스(`bg-white text-primary shadow-sm`)를 토글하고, 해당하는 기간 데이터를 `buildSalesTrendChart(period)`에 넘겨주어 Chart.js 선그래프와 막대그래프가 갱신되도록 처리.
- **도넛 차트 퍼센티지 비중 노출**:
  - `buildProductShareChart` 함수에서 Chart.js 옵션의 툴팁 콜백 함수를 활용하여 툴팁에 `%`가 표시되도록 구성하고, 범례 라벨에 명시적으로 퍼센트를 병기.
- **수금/미수금 현황 수동 업데이트 UI 연동**:
  - '거래처별 구입 누적 현황' 테이블 내의 수금 및 미수 정보 영역을 단순 텍스트 배지 대신, 사장님이 변경할 수 있는 `select` 드롭다운으로 변경.
  - 드롭다운 값을 변경하면 `BongBongStore.updateBuyerStatus`가 호출되고 로컬스토리지에 저장되어 실시간으로 반영되도록 구현.
- **KPI 실시간 주문 반영**:
  - 분석 탭 상단의 KPI 카드(총 매출액, 누적 판매량)가 고정된 텍스트 대신, 기본 과거 통계에 오늘 접수 및 마감된 발주의 합계액과 수량이 자동으로 더해져서 표시되도록 스크립트 보강.

## Verification Plan

### Automated Tests
- 없음 (정적 HTML/JS 시뮬레이터 환경)

### Manual Verification
1. **차트 기간별 전환 검증**:
   - `분석 및 통계` 탭의 '월간', '주간', '일간' 버튼을 순서대로 클릭했을 때, 우측 트렌드 차트가 해당 주기에 부합하는 라벨과 데이터로 정상 변경되는지 확인합니다.
2. **도넛 차트 비율 확인**:
   - 주요 품목별 판매 비중 차트에 마우스를 올렸을 때 툴팁에 `골드 감자: 55%`와 같이 비율이 정확히 표기되는지 확인합니다.
3. **수금/미수금 수동 변경 및 로컬 스토리지 보존 확인**:
   - 하단 테이블의 특정 거래처(예: 최유진)의 수금 상태를 `미수금 ₩120,000`에서 `정상`으로 변경합니다.
   - 페이지를 새로고침한 후에도 해당 거래처의 수금 배지가 초록색 `정상` 상태로 정상 보존되는지 확인합니다.
