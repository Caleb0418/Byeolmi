# 카카오 알림톡 켜기 — 셋업 체크리스트

> 이 문서는 **별미집 정산/발주 알림톡을 실제로 작동시키기 위한 준비 절차**입니다.
> 알림톡은 코드만으로 켜지지 않고, **카카오·Solapi 가입(사업자 인증 포함)과 비밀키 입력**이 있어야 작동합니다.
>
> 표기: 🙋 = 사장님(대표자)이 직접 / 🧑‍💻 = 개발자가 처리 / ⏳ = 카카오 검수 대기(보통 영업일 1~2일)

> ⚠️ Solapi·카카오 화면과 순서는 수시로 바뀝니다. 아래는 큰 흐름이며, **실제 진행은 각 사이트의 안내 문구를 따라** 주세요.

## 1단계. 카카오 비즈니스 채널 만들기 (🙋 사장님)

- [ ] 카카오 채널 관리자센터(`center-pf.kakao.com`) 또는 카카오 비즈니스(`business.kakao.com`)에서 **채널 개설**
- [ ] 채널을 **사업자등록 정보로 비즈니스 인증** (알림톡은 인증된 채널에서만 발송 가능)
- [ ] 대표자 **본인인증** 완료

➡️ 결과물: **카카오 채널** 1개 (인증 완료 상태)

---

## 2단계. Solapi 가입 & 발신번호 등록 (🙋 사장님)

알림톡을 실제로 쏘는 중계 서비스가 **Solapi**입니다. (이 앱이 Solapi를 사용합니다)

- [ ] `solapi.com` 회원가입 (가능하면 사장님 명의)
- [ ] **발신번호 등록** (서류 인증 필요할 수 있음 — 통신서비스 가입증명원 등)
- [ ] 결제수단 등록 (알림톡은 건당 소액 과금)

➡️ 결과물: **발신번호**(등록 완료)

---

## 3단계. Solapi ↔ 카카오 채널 연동 (🙋 사장님, 필요 시 🧑‍💻 도움)

- [ ] Solapi 콘솔에서 **카카오 채널 연동** (1단계에서 만든 채널 검색·연결)
- [ ] 연동되면 **PFID**(카카오 채널 고유 ID)가 생김 → 메모

➡️ 결과물: **PFID**

---

## 4단계. 알림톡 템플릿 2종 등록 & 검수 (🙋 사장님 + 🧑‍💻 내용 확인) ⏳

이 앱은 메시지가 **2종류**라 템플릿도 2개 등록해야 합니다.
아래 **문구 그대로** 등록하고, `#{ }` 부분은 **변수(치환문구)** 로 지정하세요.
(앱이 보내는 실제 문구가 승인된 템플릿과 일치해야 카카오가 발송을 허용합니다)

### 템플릿 ① — 정산 요청 (settlement)
```
[별미집 정산 요청 안내]

안녕하세요, #{대표자명} 대표님.
금일 별미집 도매 발주에 대한 정산 명세서가 발행되었습니다.

■ 거래 내역 요약:
#{거래내역}

■ 최종 청구 금액: ₩#{청구금액}
■ 입금 계좌: #{입금계좌}

아래 버튼을 눌러 상세 정산 명세서(송금계좌 및 금액)를 확인해 주시기 바랍니다.
```
- 버튼 1개: **웹링크(WL)** / 버튼명 `상세 정산 명세서 보기` / 링크는 발송 시 자동 주입

### 템플릿 ② — 발주 접수 확인 (order_confirm)
```
[별미집 발주 접수 안내]

안녕하세요, #{대표자명} 대표님.
요청하신 발주가 정상적으로 접수되었습니다.

■ 발주 내역:
#{발주내역}

당일 공구 마감 후 확정 단가가 적용된 정산 명세서를 다시 보내드립니다.
감사합니다.
```
- 버튼 없음

- [ ] 템플릿 ① 등록 → 검수 제출 ⏳
- [ ] 템플릿 ② 등록 → 검수 제출 ⏳
- [ ] 승인되면 각 템플릿의 **템플릿 ID** 메모 (예: `KA01TP...`)

> 💡 `#{입금계좌}` 를 변수로 두면, 나중에 진짜 계좌로 바꿔도 **템플릿 재검수 없이** 변경 가능합니다.

➡️ 결과물: **템플릿 ID 2개** (정산용 / 발주확인용)

---

## 5단계. Solapi API 키 발급 (🙋 사장님)

- [ ] Solapi 콘솔 → **API Key 관리** → 키 발급
- [ ] **API Key** 와 **API Secret** 메모 (Secret은 발급 시 1회만 보임 — 꼭 저장)

➡️ 결과물: **API Key**, **API Secret**

---

## 6단계. Supabase에 비밀값 입력 (🧑‍💻 개발자)

위에서 모은 값들을 Supabase에 입력하면 알림톡이 켜집니다.

**경로:** Supabase 대시보드 → 프로젝트(Byeolmi) → **Edge Functions → Secrets**(환경변수) → 아래 키들을 추가

| 비밀키 이름 | 넣을 값 | 출처 |
|---|---|---|
| `SOLAPI_API_KEY` | Solapi API Key | 5단계 |
| `SOLAPI_API_SECRET` | Solapi API Secret | 5단계 |
| `SOLAPI_SENDER_NUMBER` | 발신번호(숫자만, 예 01089909951) | 2단계 |
| `SOLAPI_PFID` | 카카오 채널 PFID | 3단계 |
| `SOLAPI_TEMPLATE_ID` | **정산** 템플릿 ID | 4단계 ① |
| `SOLAPI_ORDER_TEMPLATE_ID` | **발주확인** 템플릿 ID | 4단계 ② |
| `BANK_INFO` (선택) | 입금 계좌 문구 (예: `국민은행 646801-01-557728 (우모유통)`) | 미입력 시 기본값 사용 |

- [ ] 위 6개(+선택 1개) 입력 후 저장

> ℹ️ 함수 코드(`send-alimtalk`)는 **이미 배포돼 있습니다.** 비밀값만 넣으면 됩니다.
> (값을 바꾼 뒤에도 보통 자동 반영되지만, 안 되면 `supabase functions deploy send-alimtalk` 로 재배포)

---

## 7단계. 테스트 발송 (🧑‍💻 개발자 + 🙋 사장님 폰으로 확인)

- [ ] 사장님 대시보드에서 본인 번호로 **정산 알림톡 1건 테스트 발송**
- [ ] 카톡 수신 확인 (문구·버튼·계좌 정상 표기)
- [ ] 구매자 화면에서 **발주 1건** 넣어 **발주확인 알림톡** 수신 확인
- [ ] 실패 시: Supabase Edge Function **로그**에서 오류 메시지 확인 (대개 키 오타·미승인 템플릿·발신번호 미등록)

---

## 자주 막히는 곳 ⚠️

- **"Solapi API keys are not configured"** → 6단계 `SOLAPI_API_KEY/SECRET` 누락
- **발송은 됐는데 카톡이 안 옴** → 템플릿 미승인 / 템플릿 문구와 실제 문구 불일치 / PFID 오류
- **발신번호 오류** → 2단계 발신번호 등록 미완료
- **계좌를 바꾸고 싶다** → `BANK_INFO` 값만 수정(템플릿이 변수면 재검수 불필요). 화면 표시 계좌는 대시보드 **시스템 환경 설정 → 결제 계좌 및 상호 설정**에서 변경
