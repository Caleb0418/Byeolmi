# 품목 및 가격 구간 관리 기능 구현 계획서

사장님이 매일 취급하는 품목을 추가, 수정, 삭제하고 수량별 할인 단가 구간(Sliding Price Tiers)을 직접 관리할 수 있는 프론트엔드 모달 기능 구현 계획입니다.

---

## 1. 개요 및 요구사항
* **목적**: 사장님이 개발자의 도움 없이도 매일 당일 공구 품목(최대 20개 이하)을 대시보드에서 제어하고 가격 정책을 설정할 수 있게 합니다.
* **주요 기능**:
  1. **품목 목록 조회**: 현재 등록된 품목들의 이름, 카테고리, 단위, 구간 단가 설정을 대시보드에서 한눈에 모니터링합니다.
  2. **품목 추가/수정 (CRUD)**:
     * 품목명, 카테고리(신선식품/간편조리/간식/생활용품), 규격 단위(박스/팩/롤 등) 입력.
     * **동적 단가 구간 추가**: `[구간 추가]` 버튼을 눌러 기준 수량과 가격을 입력하는 행(Row)들을 동적으로 생성/삭제합니다.
  3. **품목 삭제**: 더 이상 취급하지 않는 품목은 당일 주문 내역에 영향이 가지 않도록 삭제 또는 비활성화 처리합니다.
  4. **실시간 연동**: 관리자에서 저장하는 즉시 로컬 스토리지를 거쳐 구매자용 화면(`client.html`)의 품목 리스트와 게이지바 타겟이 자동 갱신됩니다.

## 3. 검증 시나리오
* **품목 추가 테스트**: 사장님 화면에서 `[품목 및 단가 설정]` -> `[새 품목 추가]` 후 이름과 가격 구간(`0개 이상: 30000원`, `10개 이상: 25000원`) 입력 후 저장.
* **실시간 동기화 확인**: 구매자용 화면(`client.html`)을 새로고침하거나 확인할 때, 방금 추가한 신규 품목이 드롭다운과 게이지바 영역에 올바르게 노출되는지 확인.
* **구간 단가 편집 테스트**: 기존 품목의 단가 구간을 수정했을 때 대시보드 매출 예상액과 단가 적용선이 올바르게 갱신되는지 확인.
